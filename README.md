# Chinese-English Audio Data & Annotation Portfolio

This folder is ready to publish with GitHub Pages.

## Recommended GitHub Pages setup

1. Create a public GitHub repository, for example `xai-chinese-audio-portfolio`.
2. Upload all files in this folder to the repository root.
3. In GitHub, open **Settings > Pages**.
4. Set source to `Deploy from a branch`.
5. Select branch `main` and folder `/root`.
6. After GitHub publishes the site, use the public Pages URL in the PDF portfolio packet.

## File structure

- `index.html`: public-facing portfolio page
- `audio/`: five `.m4a` voice samples
- `transcripts/`: annotated transcript files
- `annotation_methodology.md`: annotation method and quality rubric
- `application_note.md`: short application note
- `portfolio_packet.pdf`: PDF version for the application upload field

## Submission strategy

Upload `portfolio_packet.pdf` to the application page. If the page or recruiter allows links, include the GitHub Pages URL so reviewers can play the audio samples directly in the browser.
